# Debugging 模式

你正在执行 /debugging 命令，需要系统性地定位和修复 bug。

## 核心原则

**不要猜测，要验证。** 每个假设必须有代码证据或运行结果支撑。

## 执行流程

### 1. 收集信息
- 使用 `read_file` 阅读相关报错文件
- 使用 `grep` 搜索错误信息和相关符号
- 使用 `bash` 查看运行日志或复现 bug
- 阅读最近的 git 提交记录了解改动历史

### 2. 建立假设
- 基于收集的信息，提出 1-3 个可能的根因假设
- 对每个假设说明证据支持度

### 3. 验证假设
- 用 `bash` 添加临时日志或断点验证
- 用 `grep` 追踪数据流确认假设
- 用 `read_file` 检查相关代码路径

### 4. 修复
- 在确认根因后，使用 `read_file` + `edit_file` 修复
- 改动要**最小化**，只修改必要的代码
- 修复后用 `bash` 验证 bug 不再出现

### 5. 回顾
- 确认没有引入新问题
- 考虑是否需要添加测试防止回归

## 输出格式

```
## Bug 描述
（现象、复现步骤、影响范围）

## 信息收集
- 错误日志/堆栈：
- 相关文件：
- 最近改动：

## 假设与验证
1. 假设A：xxx → 验证结果：xxx
2. 假设B：xxx → 验证结果：xxx

## 根因
（确切的根因，代码位置）

## 修复方案
（具体改动，文件:行号）

## 验证
（如何验证修复有效）
```
